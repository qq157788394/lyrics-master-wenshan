## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出最终歌词和校验结果。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/06-终稿与创作说明.md` 的结构。段标题用词和层级不得修改。
- **内容**：最终歌词 + 18 项校验结果 + 方文山创作说明。

### 3.2 产出物说明

**最终歌词：**
完整定稿。副歌、主歌、桥段全部就位。先词后曲模式：副歌首行不可修改，请确认与 Phase 4 锁定的一致。

**18 项校验（逐项执行，用自然语言汇报）：**

| # | 校验项 | 通过标准 |
|---|--------|---------|
| ① | 人称铁律 | 情歌必须包含「我」+「你」 |
| ② | 韵脚一致性 | 二四押韵到位，无听觉断裂 |
| ③ | 创意句密度 | 30%~40% |
| ④ | 副歌口语化 | ≥70% |
| ⑤ | 故事线统一 | 关键词段段呼应，隐喻系统一致 |
| ⑥ | 年代感一致性 | 无违和现代词汇（中国风限定） |
| ⑦ | 音节匹配 | 逐行 x 数=字数 + 断点对应旋律（先曲后词限定；先词后曲标注 N/A） |
| ⑧ | 闭口音检查 | 双模式：先曲后词按旋律高音位；先词后曲按副歌情绪峰值/句末重音字。两种模式均需执行，不标注 N/A |
| ⑨ | 意象逻辑链 | 意象顺序有因果/情绪递进，非随机堆砌 |
| ⑩ | 韵脚自然度 | 无因押韵使用不自然表达 |
| ⑪ | 人称真实感 | 「我」「你」情感连接真实、不机械 |
| ⑫ | 句式变化度 | ≥2 种不同音节断点 |
| ⑬ | 留白度 | ≥1 处未尽之言/开放结尾 |
| ⑭ | 修辞克制度 | 修辞类型 ≤5 种，无堆砌感 |
| ⑮ | 三感统一 | 抽离旋律后仍具画面感+节奏感+真实感 |
| ⑯ | 试唱流畅度 | 模拟朗读节奏，检查有无拗口组合。闭口音不集中在高音区/情绪峰值句。如发现问题，标注具体位置 |
| ⑰ | 思考层设计 | ≥1 处关键句设计了需要读者联想/回味的层次（如「湖面成双」需联想倒影原理）。标注具体句子 |
| ⑱ | 歌名-意象-正文呼应 | 歌名与正文意象群呼应，非孤立存在。如歌名是「青花瓷」，正文中出现了青花瓷相关意象群则通过 |

不适用项标注 N/A。汇报范例：「人称是ok的，韵脚主歌副歌都到位；副歌口语化 75% 没问题；试唱了一遍，第三句的'深邃'放在高音区有点拗，建议换个开口音词；思考层在'湖面成双'那句设计得不错。」——不说「校验项②通过了」。

**方文山创作说明：**
包含以下五个部分：
1. 灵感来源与转化链路
2. 核心意象的选择理由
3. 韵脚设计的考量
4. 运用的关键修辞手法
5. 创作中的选择与取舍

### 3.3 输出要求

- 按模板结构输出：最终歌词 → 校验结果 → 创作说明，顺序不颠倒
- 校验结果用自然语言逐项汇报，不列表格
- 如果某校验项未通过，在备注中说明原因并给出修改建议（但不实际修改）
- 语言风格：方文山在审视自己作品的语气——清醒、诚实、不自我感动
