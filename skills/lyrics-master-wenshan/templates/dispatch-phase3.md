## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出骨架内容。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/03-歌词骨架.md` 的结构。段标题用词和层级不得修改。
- **内容**：一段完整的段落蓝图，填满所有实际使用的段落行。

### 3.2 产出物说明

段落蓝图表，每行一个段落，填写以下 9 列：

| 列 | 填写规则 |
|----|---------|
| 段落 | 按歌曲实际播放顺序列出。A1→A2→Pre-C(如有)→B→A3(如有)→Pre-C(如有)→B（重复）→Inter(如有)→Bridge(如有)→B（重复）。先曲后词模式 A3/A4 依 x 标签决定。**A = 主歌，B = 副歌，Bridge = 桥段（C）。副歌歌词原则上相同，B 首次出现时填写全部决策列，后续「B（重复）」仅标注位置** |
| 行数×字数 | 先词后曲：自定音节断点（遵守行数一致、前两句字数一致；设计非对称断点如「三四三五」避免呆板，脑中需有旋律节奏）。先曲后词：从用户 x 结构逐行读取 |
| 押韵 | 韵类 + 押韵方式。二四押韵为基准，一二四韵可选变体，禁止一三四韵。主副歌可换韵。罕见韵脚占比 ≤30% |
| 人称 | 每段的人称使用策略（如「第三人称铺陈→A2 转入'我'→B'你'首现→A3'我''你'交替」） |
| 画面 | 主歌为主标注画面元素，副歌可略 |
| 态度 | 叙事者立场：旁观 / 代入 / 质问 / 接受 |
| 情感 | 每段的情感核心（爱/恨/遗憾/怀念），情歌必须始终贯穿「我」+「你」。先词后曲模式下，第30天指出「意境」可替代「情感」——如《告白气球》偏重旋律则情感可弱化 |
| 情绪 | 能量级标注（低 → 中 → 高），整体形成情绪弧线 |
| 故事线 | 叙事推进节点（如「初见→离别→思念」），与 Phase 2 的概念协调一致 |

先曲后词模式额外规则：用户 x 标签 `[Verse]`→A1/A2/A3…、`[Pre-Chorus]`→Pre-C、`[Chorus]`→B、`[Bridge]`→Bridge。**注意：`[Final-Chorus]` 在结构推断时等同于 `[Chorus]`**，蓝图表中标注为「B（重复）」。行数×字数直接取自 x 数量，无需重算。`[Intro][Interlude][Outro]` 不计入蓝图表。

**结构推断规则**：从 x 标签排列顺序直接推断结构名称。完整规则如下：

| x 标签序列 | 推断结构 | 蓝图表标注 |
|-----------|---------|-----------|
| Verse→Verse→Chorus→Verse→Chorus→Chorus | ABABB | A1/A2/B/A3/B(重复)/B(重复) |
| Verse→Verse→Chorus→Chorus | ABAB | A1/A2/B/B(重复) |
| Verse→Chorus→Verse→Chorus→Bridge→Chorus | ABABCB | A1/B/A2/B(重复)/Bridge/B(重复) |
| Verse→Pre-Chorus→Chorus→Verse→Pre-Chorus→Chorus→Bridge→Chorus | ABABCB（含导歌） | A1/Pre-C/B/A2/Pre-C/B(重复)/Bridge/B(重复) |
| Verse→Chorus→Verse→Chorus | ABAB | A1/B/A2/B(重复) |

**关键认知**：ABABB 的本质是「两段主歌 + 一段副歌」重复一次后，最后再来一段副歌收尾。**核心规则**：Verse 按出现顺序编号 A1/A2/A3…，B 不编号（副歌歌词原则上相同），仅首次出现标注为 B，后续重复出现标注为「B（重复）」。结构名称仍使用 ABABB / ABABCB 等（A 段数 + B 段数 + 是否有 Bridge）。将推断的结构名称填入 03 模板头部。

先词后曲模式额外规则：歌曲结构（ABABCB 或 ABABB）和导歌有无已在 Phase 3 开头由主 Agent 向用户确认。按确认的结构填表，不用的行（如无导歌则删 Pre-C 行、无桥段则删 Bridge 行）整行移除。追加加载第30天。

{如果用户是中国风} 韵脚规划时注意配合仿古词汇的韵律感，情绪走含蓄典雅路线。追加加载第03天、第05天。

- 语言风格：像方文山在交代一首歌的结构设计，简洁自信
