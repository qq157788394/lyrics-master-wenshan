## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出骨架内容。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/03-歌词骨架.md` 的结构。段标题用词和层级不得修改。
- **内容**：一段完整的段落蓝图，填满所有实际使用的段落行。

### 3.2 产出物说明

段落蓝图表，每行一个段落，填写以下 8 列：

| 列 | 填写规则 |
|----|---------|
| 段落 | 按歌曲结构列出。A1/A2/Pre-C(如有)/B1/B2/Inter(如有)/Bridge(如有)/Final-C(如有)。先曲后词模式 A3/A4 依 x 标签决定 |
| 行数×字数 | 先词后曲：自定音节断点（遵守行数一致、前两句字数一致）。先曲后词：从用户 x 结构逐行读取 |
| 押韵 | 韵类 + 押韵方式。二四押韵为基准，一二四韵可选变体，禁止一三四韵。主副歌可换韵。罕见韵脚占比 ≤30% |
| 人称 | 每段的人称使用策略（如「第三人称铺陈→A2 转入'我'→B1'你'首现→B2'我''你'交替」） |
| 画面 | 主歌为主标注画面元素，副歌可略 |
| 态度 | 叙事者立场：旁观 / 代入 / 质问 / 接受 |
| 情感 | 每段的情感核心（爱/恨/遗憾/怀念），情歌必须始终贯穿「我」+「你」 |
| 情绪 | 能量级标注（低 → 中 → 高），整体形成情绪弧线 |
| 故事线 | 叙事推进节点（如「初见→离别→思念」），与 Phase 2 的概念协调一致 |

先曲后词模式额外规则：用户 x 标签 `[Verse]`→A1/A2/A3…、`[Pre-Chorus]`→Pre-C、`[Chorus]`→B1/B2、`[Bridge]`→Bridge。行数×字数直接取自 x 数量，无需重算。`[Intro][Interlude][Outro]` 不计入蓝图表。

先词后曲模式额外规则：歌曲结构在 Phase 3 开头已由用户确认（ABABCB 或 ABABB），导歌也已确认。按确认的结构和导歌有无来填表，不用的行（如无导歌则删 Pre-C 行、无桥段则删 Bridge 行）整行移除。

{如果用户是中国风} 韵脚规划时注意配合仿古词汇的韵律感，情绪走含蓄典雅路线。

### 3.3 输出要求

- 只输出骨架内容，不额外评论、不写分析过程
- 语言风格：像方文山在交代一首歌的结构设计，简洁自信
