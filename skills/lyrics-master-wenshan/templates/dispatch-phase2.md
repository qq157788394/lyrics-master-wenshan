## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出方案内容。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/02-概念方案.md` 的结构。段标题（## 方案一、## 方案二、## 方案三、## 演唱者适配）用词和层级不得修改。
- **内容**：3 个不同的创作方向，不可多不可少。

### 3.2 产出物说明

每个方案包含以下字段：

| 字段 | 要求 |
|------|------|
| 方向标签 | 一句话概括本方案的气质 |
| 歌名 | 从歌名创作四策略（长度优势 / 动作加持 / 矛盾组合 / 极简主义）中至少使用两种不同策略。注意通俗性：放弃过于晦涩的词汇，选择大众熟悉但仍有新鲜感的词（如「青花瓷」优于「汝窑」）。若原名全是名词缺乏情感张力，加入情感词（如「美索不达米亚」→「爱在西元前」）。歌名可能在 Phase 4 副歌写完后微调，当前先作为创作方向锚点 |
| 核心意象 | 三元素构图法（3 个具象名词定画面基调）。中国风须用年代符号重构法选物品，意象间有因果递进，不堆砌 |
| 情感色调 | 从情感最大公约数中选取（失恋 87%、暗恋 92% 等大众经验） |
| 一句话预览 | 让用户感知这首歌的气质 |
| 词义方向 | 对照曲风→词义匹配表确定 |

方案间差异须显著——不同歌名策略、不同意象群、不同情感色调。不可换词不换骨。

演唱者适配：
- 男声：「我」的男性角色感融入意象和语气
- 女声：「我」的女性角色感融入意象和语气
- 对唱：每个方案标注男女视角分配

{如果用户是中国风} 中国风的意象须通过三大判准验证。古今词汇交错（如「你的笑容/泛黄」），不全篇古文。

{如果是影视/游戏主题曲} 每个方案的歌名须包含剧名/游戏关键词。追加加载第24天（影视）/ 第25天（游戏）。

{如果是命题/活动专案} 追加加载第26天。

{如果用户关注风格塑造或创作风格} 追加加载第20天。

### 3.3 输出要求

- 只输出方案内容，不额外评论、不写分析过程
- 语言风格：像方文山跟朋友聊歌，不用术语轰炸用户
