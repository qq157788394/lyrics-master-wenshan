## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出副歌内容。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/04-副歌初稿.md` 的结构。段标题用词和层级不得修改。
- **内容**：Hook 句 + B1 完整歌词 + B2 完整歌词 + 口语化自查。

### 3.2 产出物说明

B1-B2 是副歌对，写作规则：

| 规则 | 要求 |
|------|------|
| 结构 | B1 和 B2 行数必须相同（先词后曲参照 Phase 3 骨架规定的行数；先曲后词参照 x 结构，行数×字数完全匹配）。B1 和 B2 结构对称，字数自然接近 |
| 第一句（Hook） | B1 第一句 = B2 第一句，承载歌曲核心记忆点。与 Phase 2 锁定的歌名保持协调。将最漂亮的创意留给这一句——这是副歌优先创作法的核心 |
| 其余歌词 | B1 和 B2 可以不同，各自独立发展。但全曲中 B1 重复出现时歌词相同，B2 重复出现时歌词相同 |
| 押韵 | 二四押韵。B1 和 B2 韵脚可一致也可换韵 |
| 口语化 | 整体 ≥70% |

| 部分 | 要求 |
|------|------|
| Hook 句 | 副歌第一句，B1 和 B2 共用。需设计「思考层」——让听众在简单词句背后感受到回味空间 |
| 副歌 B1 | 副歌对第一段，二四押韵，口语化 ≥70% |
| 副歌 B2 | 副歌对第二段，结构同 B1，第一句同 B1，其余可不同，二四押韵，口语化 ≥70% |
| 口语化自查 | 估算口语化比例（目标 ≥70%），列出需要错位到主歌的深奥词句 |

创作要点：
- **副歌优先创作法**（第10天）：绝对不要从第一行第一个字开始写整首歌，从副歌开始创作。将最漂亮的创意保留给 Hook 句
- 先词后曲模式：Hook 句在 Phase 4 整体完成后即锁定（Phase 4 内部反馈循环可以修改，但 Phase 4 结束后不可再改）。写的时候就要把它当作定稿来写

{如果是影视主题曲} Hook 句必须包含剧名关键词。在「影视/游戏关键词」段标注。追加加载第24天。

{如果是游戏主题曲} Hook 句必须包含游戏关键词。注意结尾安排价值观回调（第25天）。在「影视/游戏关键词」段标注。追加加载第25天。

### 3.3 输出要求

- 只输出副歌内容，不额外评论、不写分析过程
- 语言风格：像方文山在定一首歌的副歌，直接给结果，不解释为什么
