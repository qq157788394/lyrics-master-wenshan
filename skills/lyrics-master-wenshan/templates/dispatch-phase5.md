## 三、交付目标

### 3.1 产出物

- **形式**：在对话中直接输出完整歌词。**不要写文件**——文件由主 Agent 负责落盘。
- **格式**：严格按 `{SKILL_ROOT}/templates/05-完整歌词.md` 的结构。段标题用词和层级不得修改。
- **内容**：完整歌词（Intro→A1→A2→Pre-C→Chorus→Interlude→A3→Pre-C→Chorus→Bridge→Chorus→Outro）+ 画面-情感配比 + 创作备注。

### 3.2 产出物说明

| 部分 | 要求 |
|------|------|
| 完整歌词 | 包含所有段落标签 `[Intro][Verse][Pre-Chorus][Chorus][Interlude][Bridge][Outro]`。`[Chorus]` 在歌曲中重复出现，歌词原则上相同，如有变体需标注。副歌直接引用 Phase 4 已锁定的内容，**不修改副歌首行**。先词后曲模式：副歌首行在 Phase 4 结束后已锁定，此处仅做引用，不可改动任何字词 |
| 画面-情感配比 | 按段落级统计（非全曲汇总）。每 4 句画面后 1~2 句情感，目标 3:2。主歌偏画面，副歌偏情感 |
| 创作备注 | 统计使用的修辞类型及数量（目标 ≤5 种），标注去 AI 味处理（意象逻辑链 / 韵脚自然度 / 人称真实感 / 句式变化度 / 留白度） |

创作要点：
- 画面序列按点→线→面层次推进，意象间有因果递进
- 中国风须用年代符号重构法，所有物品属于同一时代
- 句式长短交替，同一段内至少 2 种不同音节断点
- 主歌末尾留空间给导歌或副歌，不自行把情绪峰值用完
- 至少留一处未尽之言或开放结尾
- 对唱模式：按 Phase 3 骨架的人称列分配男女段，设计视角差异
- 修辞服务于意境，不为修辞而修辞
- **质量分布警告**（第10天）：切忌将精彩语句都放在主歌而副歌用语浅显无聊。主歌为副歌做铺垫，但不可在文采上压过副歌
- **重复结构战法**（第19天）：音乐段落过近时，通过重复特定字词建立文本韵律（如「在」字重复：「在旁/在想/在看」），增强主歌与导歌的衔接
- **主歌-副歌衔接检查**：完成初稿后，自查主歌末尾的情绪和意象是否顺畅过渡到副歌首行。如果主歌结尾的意象与副歌 Hook 句之间缺乏情绪递进，调整主歌末句

{如果用户是中国风} 仿古词汇与古今词汇交错使用，不得出现违和现代词汇。可考虑首尾呼应闭环结构（如适合）。

### 3.3 输出要求

- 只输出歌词内容 + 配比统计 + 创作备注，不额外评论
- 语言风格：像方文山在交一份完整的词，简洁自信
